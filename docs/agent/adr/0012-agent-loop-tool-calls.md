# ADR-0012: bounded agent-turn loop for `/agent` (M11)

## Context

User dogfood feedback on 2026-05-12 identified a product mismatch:
`/agent` was labelled as an agent surface, but the shipped behaviour was
still **single prompt → single completion → done**. Users expected the
page to investigate with tools, not only explain what command they could
run themselves.

Studio already had the pieces needed to close most of that gap without
waiting for MCP adapters or provider-native tool-call support:

- `studio-router` can run repeated completions.
- `studio-server` already owns project-root scoping, ledger plumbing, and
  SSE delivery.
- the SvelteKit `/agent` page can render an event timeline naturally.

The question for M11 was not “full external-agent parity or nothing”, but
whether to ship an **honest bounded vertical slice** that makes `/agent`
useful now.

## Decision

Accept and ship a **bounded agent-turn loop** in Studio itself.

The local M11 slice is intentionally narrower than the original spike:

- Studio adds a new `POST /api/agent-turn` route.
- The loop is bounded (`max_iterations`, default 16, hard cap 16).
- Tools are implemented by `studio-server` and scoped to `--project`.
- Tool calling uses a **text-JSON protocol embedded in normal model
  completions**, not provider-native function/tool APIs.
- `/agent` becomes an **iteration timeline** UI.
- model selection is initialized from model-discovery routes instead of a
  hardcoded UI default.

This is sufficient to close the “one-shot dispatch disguised as agent” UX
problem while keeping the safety surface understandable.

## Shipped scope

### 1. `POST /api/agent-turn`

Request body:

```json
{
  "model": "<required>",
  "system": "optional system prompt",
  "messages": [{ "role": "user", "content": "..." }],
  "params": { "temperature": 0.2 },
  "max_iterations": 16,
  "tools_allowed": ["fs.read", "fs.list", "git.status", "git.diff", "project_tree"],
  "task_tag": "agent-turn"
}
```

Validation:

- `model` must be non-empty.
- `messages` must contain at least one entry.
- `max_iterations` must be `1..=16`.
- `tools_allowed` must be a known allow-list subset.
- write/exec tools are rejected unless `--enable-write-tools` is set.
- `task_tag` reuses ADR-0010 validation (`<= 256` bytes, no control
  chars, empty string normalises to `None`).

### 2. SSE event contract

`/api/agent-turn` streams these event types:

- `iteration`
- `tool_call`
- `tool_result`
- `done`
- `error`

Example shape:

```text
event: iteration
data: {"n":0,"model":"...","text":"...","usage":{...},"cache_hit":false,"stop_reason":"tool_use"}

event: tool_call
data: {"iteration":0,"tool":"fs.list","input":{"dir":"/"}}

event: tool_result
data: {"iteration":0,"tool":"fs.list","output":{"dir":"/","entries":[...]},"error":null,"ms":3}

event: done
data: {"final_text":"...","iterations":2,"total_tokens":{"prompt_tokens":123,"completion_tokens":45},"task_tag":"agent-turn"}
```

`error` is terminal for router/turn failures. `done` is terminal for both
normal completion and max-iteration exhaustion (`final_text` becomes
`"max iterations reached"`).

### 3. Tool protocol: text JSON, not native provider tools

The server prepends a system message generated by
`tool_protocol_message(...)` telling the model to respond with one of two
JSON envelopes:

```json
{"tool_calls":[{"tool":"fs.list","input":{"dir":"/"}}]}
```

or

```json
{"final_text":"your answer"}
```

`parse_agent_directive(...)` accepts:

- plain JSON,
- fenced JSON,
- or JSON embedded inside surrounding text.

If parsing fails, Studio treats the response as final assistant text.

This means the shipped M11 slice is **provider-agnostic over normal text
completions**, but it does **not** yet claim provider-native function
calling, MCP, or any standard tool schema handshake.

### 4. Built-in tools and safety policy

**Default tools (read-only):**

- `fs.read`
- `fs.list`
- `git.status`
- `git.diff`
- `project_tree`

**Opt-in tools (`--enable-write-tools` required):**

- `fs.write`
- `fs.delete`
- `shell.exec`

Safety properties implemented in the local slice:

- all filesystem paths are resolved relative to `--project` and rejected
  on traversal / escape.
- `fs.read` is capped at 1 MiB and UTF-8 only.
- `project_tree` is capped and skips heavyweight dirs such as `.git`,
  `target`, `node_modules`, `.svelte-kit`.
- `fs.delete` refuses to delete git-tracked files.
- `shell.exec` refuses `sudo`, defaults to project-root cwd, and clamps
  timeout to at most 5 minutes.
- write/exec tools are denied at request-normalisation time when the boot
  flag is absent.

### 5. Model discovery replaces hardcoded UI defaults

M11 also ships non-secret model discovery routes:

- `POST /api/models/preview` — uses the endpoint/key entered on `/login`
  without persisting them.
- `GET /api/models/session` — lists models from the authenticated sealed
  session and returns the session-selected model.

Frontend behaviour:

- `/login` can fetch models from the entered provider before submit.
- `/agent` first checks session status, then seeds its model input from
  `selected_model` and `/api/models/session`.
- the `/agent` page no longer hardcodes `claude-opus-4-7` as the UI
  default.

### 6. `/agent` becomes a timeline, not a one-shot composer

The page now renders a bounded turn as a chronological timeline of:

- iteration cards,
- tool call cards,
- tool result cards,
- final text,
- and terminal error state when present.

The composer still uses a single user prompt and optional system prompt,
but the output model is now “investigate over iterations” rather than
“single completion stream”.

## Explicitly not shipped in this slice

The accepted M11 slice does **not** include:

- provider-native function/tool calling APIs,
- MCP server integration,
- subagent spawning,
- nested multi-agent timelines,
- background commands,
- network tools,
- or any claim that `/api/dispatch` has changed.

`/api/dispatch` remains the one-shot completion route.

## Consequences

### Positive

- `/agent` now matches its name much more closely.
- Studio gets an honest bounded agent surface without waiting for larger
  adapter work.
- the safety model is understandable: read-only by default, write/exec
  opt-in at boot.
- model selection is driven by the configured provider/session rather than
  a stale frontend constant.

### Trade-off

- the text-JSON protocol is less robust than native provider tool-call
  APIs and relies on prompt adherence plus tolerant parsing.
- the slice is intentionally single-agent and bounded; it is a useful
  vertical slice, not yet a general external-agent platform.

## Follow-on work left open

Deferred beyond this accepted slice:

- provider-native tool-call translation,
- MCP adapters,
- richer tool schemas,
- browser e2e coverage for the new `/agent` timeline,
- and any orchestrator/subagent model.

Those should land only when the implementation exists; this ADR does not
pre-approve them.
